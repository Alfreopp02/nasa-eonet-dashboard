# 📋 Specifica del Test Pratico 
TEST PRATICO DEVELOPER 
Obiettivo ed istruzioni del Test: 
L’obiettivo del presente test è valutare la capacità del candidato di progettare e implementare un 
sistema completo in grado di recuperare, gestire, analizzare e visualizzare dati provenienti da 
fonti pubbliche NASA (API), con particolare riferimento alle missioni spaziali di maggiore rilevanza 
scientifica o mediatica. 
Il candidato dovrà realizzare una dashboard interattiva, capace di facilitare la consultazione dei dati 
e l’individuazione di pattern o insight significativi. La soluzione dovrà inoltre prevedere funzionalità 
di filtro basate su periodo, tipologia di missione o area geografica. 
Il progetto dovrà essere versionato e pubblicato su GitHub, includendo: 
• Codice sorgente completo   
• Database utilizzato   
• Documentazione tecnica   
Sono inoltre richieste riflessioni personali riguardanti le scelte progettuali effettuate e sulle possibili 
evoluzioni future del sistema. 
Scenario: 
La NASA mette a disposizione un’ampia gamma di API pubbliche tramite il portale ufficiale 
https://api.nasa.gov. Tali API consentono l’accesso a dati relativi a missioni spaziali, immagini 
satellitari, osservazioni astronomiche, asteroidi e numerose altre risorse scientifiche. 
Il compito consiste nel progettare un flusso dati che, sfruttando una o più di queste API, raccolga e 
organizzi le informazioni sulle missioni più rilevanti, rendendole successivamente fruibili tramite 
una dashboard interattiva e facilmente navigabile. 
Obiettivi del Progetto: 
Il candidato dovrà realizzare un sistema che soddisfi i seguenti punti: 
• Recupero dei dati da una o più API pubbliche NASA.   
• Persistenza dei dati all’interno di un database relazionale (SQL) progettato e documentato.   
• Generazione di un output informativo sotto forma di dashboard, grafici, report o tabelle 
interattive, evidenziando correlazioni o trend di interesse.   
• Filtraggio avanzato dei dati per periodo, tipologia di missione o area geografica. 
Requisiti Tecnici Minimi: 
• Scrittura manuale di query SQL (CRUD e almeno una query analitica con JOIN o funzioni 
aggregate).   
• Integrazione con una o più API NASA tramite linguaggio di sviluppo a scelta.   
• Progettazione di uno schema dati coerente, documentato tramite diagramma ER o 
equivalente.   
• Produzione di un output finale chiaro e di immediata consultazione. 
Bonus Facoltativi: 
Sono considerati elementi aggiuntivi meritevoli: 
• Tecniche di caching e/o normalizzazione per ottimizzare le prestazioni.   
• Visualizzazioni avanzate: analisi temporali, correlazioni, mappe geografiche, modelli 
predittivi.   
• Interfaccia utente curata, ordinata ed efficace.   
• Documentazione dettagliata che illustri scelte progettuali, criticità riscontrate e potenziali 
miglioramenti. 
Richieste pre-inizio lavori: 
Viene richiesta prima dell’inizio dei lavori: 
• una stima di tempo 
• una data di consegna prima dell’effettivo inizio dei lavori 
Consegna Finale: 
La consegna dovrà includere: 
1. Codice sorgente del progetto.   
2. Schema SQL del database utilizzato.   
3. Relazione tecnica che descriva:   
a. API selezionate e motivazioni della scelta   
b. Architettura della soluzione   
c. Query principali   
d. Insight ricavati dall’analisi dei dati   
4. Repository GitHub completo e navigabile. 

Elementi Personali e di Originalità:
Riflessione Personale e Contestuale 
Il candidato dovrà allegare un breve aneddoto o una riflessione che descriva una situazione reale in 
cui ha lavorato con dati esterni e API pubbliche, evidenziando come tali esperienze abbiano 
influenzato il processo di progettazione.

Approccio Innovativo:
Presentare almeno un’idea o proposta migliorativa che possa rendere il progetto più efficace o 
distintivo, ad esempio tramite automazioni, sistemi di alert, algoritmi di analisi avanzata o tecniche 
di data enrichment. 

Tocco Creativo:
Si richiede una personalizzazione dell’aspetto grafico della dashboard, con elementi che rendano 
l’esperienza utente gradevole, chiara e intuitiva.
